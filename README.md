# grist-search
search widget for grist : created for Mastercraft.

Problem assigned: 

President has all information regarding architects, general contractors, school boards, etc (thousands of entries) on paper, in gigantic binder. Currently, to find information about some project or data, she has to sort through hundreds and hundreds of pages to find the information she needs. Additionally, to update information, she handwrites all her data onto those papers.

This information, however, is also put into a spreadsheet via grist through another employee. The current process is: 
1. president gets work order
2. president searches for general contractor name, through the binder, to see if job should be approved.
3. If job is approved, president updates project specifics on that general contractors file, and passes that file to assistant.
4. Assistant takes that information and updates spreadsheet so other online softwares, such as orderings, are synced to project requirements.

I was tasked to make viewing and updating this spreadsheet as easy as possible (president is not technically savy).

To accomplish this task, I learned the structures of how grist works, and updated the format of some spreadsheets to make data entry easier. I then vibecoded a custom widget via Claude to meet the exact specifications that the president asked for as a way to view the data.
